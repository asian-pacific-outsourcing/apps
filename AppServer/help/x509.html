Secured Server Operation

 

Overview

For secure operation, the Application Server authenticates itself to clients using Public Key Infrastructure (PKI) credentials in the form of X509Certificates. You must create the server's credentials before attempting to run the server or clients with the x509 command line option.

The server is currently hardcoded to look for its credentials in a file called "testkeys". This file must be placed in the same directory as the server's jar file.

The easiest way to create the SSL/TLS credentials is to use the Java keytool. The keytool.exe program is found in the bin subdirectory of the JDK installation directory. A shortcut may be created to this program, or the directory may be placed in the machine's path variable. An example follows.

Creating an X509 Certificate

This example shows how to create credentials for a site, "spank.apo.com" owned by Asia Pacific Outsourcing, located in Hong Kong. Obviously these values need to change!!

Note: User responses are emboldened and Enter represents your end-of-line key.

% keytool -genkey -keyalg rsa -keystore testkeys -alias widgets
Enter keystore password: passphrase
What is your first and last name?
[Unknown]: spank.apo.com Enter
What is the name of your organizational unit?
[Unknown]: Customer Spank Group Enter
What is the name of your organization?
[Unknown]: Asia Pacific Outsourcing Enter
What is the name of your City or Locality?
[Unknown]: Hong Kong Enter
What is the name of your State or Province?
[Unknown]: HK Enter
What is the two-letter country code for this unit?
[Unknown]: HK Enter
Is CN=spank.apo.com, OU=Customer Spank Group, O="Asia Pacific Outsourcing", L=Hong Kong, ST=HK, C=HK correct?
[no]: yes Enter
Enter key password for (RETURN if same as keystore password): Enter

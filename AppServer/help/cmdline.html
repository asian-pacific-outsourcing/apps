Command Line Options

The server and client applications can be configured via optional arguments specified on the command line. If no arguments are specified, then the program will simply run using default settings.

How To Specify Command Line Options

Specifying command line arguments is operating system specific, but you essentially type the options after the target command. For Windows this is most easily accomplished by creating a shortcut to the program then modifying its properties:

  1. Right click on the shortcut, and choose Properties
  2. In the Target field, enter the desired arguments
Note 1: Be sure to leave a space between each argument as well as between the command and the first argument.
Note 2: Be sure to not leave any spaces between the colon (:) separating the option name and its value.

Available Command Line Options

In the following list, some commands take a value, these are prefixed with a hyphen and contain a colon separating the command from the value. For example -u:rick is the user command with a value of rick. It will automatically cause the user field of the login screen to with "rick".

Other commands turn on a certain behavior: These command have no hyphen or colon.

  • -u:value Sets the user login name. Defaults to blank.
  • -port:value Sets the server port. If the server is started with this command, then all of the clients must also specify the same port value. Defaults to port 8080.
  • -mode:value This is application specific:
    • For the application server it specifies the server mode.
    • For applications like TQ and VO, where the employee follows a script, the value specifies the script. When this option is not specified, the contact fields screen is used instead. However, this functionality is not fully implemented or released.
    • For DM, the mode is only used for debugging.
  • x509 Enables X509 certificate security.
  • loopback Runs the server as a localhost (IP: 127.0.0.1) for development and debugging.
  • remote For client applications, causes the application to access the application server via the Internet. For this option to work correctly, port forwarding must be enabled on the router, as set up by JP
